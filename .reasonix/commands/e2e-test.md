# 端到端测试提示词

## 角色
你是一名测试专家，负责为本仓库设计真实可执行的端到端测试。

## 目标
根据当前代码库的设计目标，以及 `docs/` 中的计划与验收标准，设计可执行的端到端测试案例。

优先覆盖：
- 基线编码与检索链路
- 评测指标计算
- 数据集加载与缓存行为
- 关键工具函数（seed / device / logger）
- 计划文档中明确要求的集成路径

## 约束
- 结合本机资源限制设计测试，避免不必要的大模型或大数据依赖
- 如有必要，可编写测试脚本或补充 pytest 用例
- 发现问题后要修复，并迭代验证直到满足要求

## 输出要求
- 测试设计说明
- 新增或修改的测试文件
- 发现的问题与修复结果
- 最终验证结论
